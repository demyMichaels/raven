Regarding ML Models & OCR
Since this is a frontend-only canvas app, I've simulated the intelligent features:

Appliance Detection: The app uses a device profile database with typical power ratings. In a production system, you would integrate with smart plugs or use disaggregation algorithms (like NILMTK) to detect appliances from aggregate readings.

OCR for Meter Reading: This would require a backend service using Tesseract OCR or cloud APIs (Google Vision, AWS Textract). The app is structured to accept meter readings manually for now.

Pattern Recognition: The simulation generates realistic hourly/daily patterns. In production, you'd use time-series analysis (ARIMA, Prophet) or simple ML models (scikit-learn) for predictions.
