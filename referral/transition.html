What Are CSS Transitions?

CSS transitions allow you to change property values smoothly over a specified duration. Instead of instant changes, transitions create fluid animations that enhance user experience and provide visual feedback.

The Transition Syntax

transition: property duration timing-function delay;

/* Examples */
transition: all 0.3s ease;
transition: transform 0.15s cubic-bezier(0.4, 0.0, 0.2, 1);
transition: box-shadow 0.28s cubic-bezier(0.4, 0.0, 0.2, 1),
            transform 0.15s cubic-bezier(0.4, 0.0, 0.2, 1);

Property

Which CSS property to animate (e.g., transform, opacity, or all)

Duration

How long the transition takes (e.g., 0.3s, 200ms)

Timing Function

The acceleration curve (e.g., ease, linear, cubic-bezier())

Delay

Optional delay before starting (e.g., 0.1s)

Best Practices & Tips

âœ“

DO: Keep it Fast

Use durations between 150-300ms for most UI interactions. Material Design recommends 0.15s for transforms and 0.28s for shadows.

âœ—

DON'T: Animate Everything

Avoid using transition: all on complex elements. It can cause performance issues.

âœ“

DO: Use Transform & Opacity

Transform and opacity are hardware-accelerated. They're perfect for smooth 60fps animations without repaints.

âœ—

DON'T: Overdo It

Too many animations can overwhelm users. Use transitions to enhance UX, not distract from it.

âœ“

DO: Provide Feedback

Use transitions to acknowledge user actions. Hover states, click effects, and state changes should be smooth.

âœ“

DO: Test Performance

Open DevTools and check the Performance tab. Aim for 60fps. Use will-change sparingly.
