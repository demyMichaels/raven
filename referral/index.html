Material Elevation in FinTech

Explore how Material Design's 10 elevation levels create hierarchy and depth in modern banking apps

üì± Why Elevation Matters in Finance

Material Design uses elevation (depth via shadows) to establish visual hierarchy, helping users understand which elements are interactive, important, or temporary.

In fintech apps, proper elevation guides users through complex flows like payments, transfers, and account management while maintaining trust through clear, predictable interactions.

üí° Pro Tip: Higher elevation = higher priority or more temporary. Permanent UI (like nav bars) uses moderate elevation, while modals and dialogs use maximum elevation.

üí° Best Practices for FinTech Apps

‚úÖ Do

  • ‚Ä¢ Use consistent elevation for similar components
  • ‚Ä¢ Higher elevation = higher priority or temporary state
  • ‚Ä¢ Limit elevation changes to maintain visual stability
  • ‚Ä¢ Use elevation to guide users through payment flows
  • ‚Ä¢ Adjust shadow opacity for dark mode (lighter shadows)

‚ùå Don't

  • ‚Ä¢ Don't use too many elevation levels at once
  • ‚Ä¢ Don't use high elevation for permanent UI elements
  • ‚Ä¢ Don't skip elevation levels arbitrarily (1 ‚Üí 12 ‚Üí 24)
  • ‚Ä¢ Don't make critical actions have low elevation
  • ‚Ä¢ Don't use the same elevation for overlapping elements
