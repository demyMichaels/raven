Material Design Card Transitions

Learn the motion system behind Material Design's beautiful card animations

Understanding the Basics

Material Design uses cubic-bezier timing functions to create natural, responsive animations. The standard transition combines two properties:

  • box-shadow (0.28s) - Creates depth perception
  • transform (0.15s) - Provides physical movement
transition: box-shadow 0.28s cubic-bezier(0.4, 0.0, 0.2, 1),
transform 0.15s cubic-bezier(0.4, 0.0, 0.2, 1);

Duration Guidelines

Property Duration Reason
box-shadow 0.28s Depth changes need more time to be perceptible
transform 0.15s Position changes should feel snappy and responsive
opacity 0.15s Quick fade transitions
color/background 0.15s - 0.3s Smooth color transitions without being sluggish

Complete Code Examples

Basic Material Card

.material-card {
  /* Default elevation (2dp) */
  box-shadow:
    0 2px 1px -1px rgba(0,0,0,.2),
    0 1px 1px 0 rgba(0,0,0,.14),
    0 1px 3px 0 rgba(0,0,0,.12);

  /* Standard Material Design transitions */
  transition:
    box-shadow 0.28s cubic-bezier(0.4, 0.0, 0.2, 1),
    transform 0.15s cubic-bezier(0.4, 0.0, 0.2, 1);
}

.material-card:hover {
  /* Elevated state (8dp) */
  box-shadow:
    0 5px 5px -3px rgba(0,0,0,.2),
    0 8px 10px 1px rgba(0,0,0,.14),
    0 3px 14px 2px rgba(0,0,0,.12);

  transform: translateY(-2px);
}

Using CSS Variables (Recommended)

:root {
  --md-ease-standard: cubic-bezier(0.4, 0.0, 0.2, 1);
  --md-ease-decelerate: cubic-bezier(0.0, 0.0, 0.2, 1);
  --md-ease-accelerate: cubic-bezier(0.4, 0.0, 1, 1);
  --md-ease-sharp: cubic-bezier(0.4, 0.0, 0.6, 1);

  --md-duration-short: 0.15s;
  --md-duration-medium: 0.28s;
  --md-duration-long: 0.375s;
}

.card {
  transition:
    box-shadow var(--md-duration-medium) var(--md-ease-standard),
    transform var(--md-duration-short) var(--md-ease-standard);
}

Best Practices

‚úÖ Do

  • ‚Ä¢ Use cubic-bezier timing functions for natural motion
  • ‚Ä¢ Keep transform durations shorter than shadow durations
  • ‚Ä¢ Use will-change: transform for performance on animated elements
  • ‚Ä¢ Test on mobile devices for smooth 60fps animations
  • ‚Ä¢ Respect prefers-reduced-motion for accessibility

‚ùå Don't

  • ‚Ä¢ Use linear timing - it feels robotic
  • ‚Ä¢ Make transitions too slow (>400ms for small UI)
  • ‚Ä¢ Transition properties like width/height (use transform instead)
  • ‚Ä¢ Use transition: all (specify exact properties for performance)
  • ‚Ä¢ Forget about dark mode shadow adjustments

Accessibility Considerations

Always respect user preferences for reduced motion: 

@media (prefers-reduced-motion: reduce) {
  .material-card {
    transition: none !important;
    transform: none !important;
  }
}

/* Or use shorter durations */
@media (prefers-reduced-motion: reduce) {
  .material-card {
    transition-duration: 0.01s !important;
  }
}
