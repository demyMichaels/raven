Best Practices for EdTech Mobile Apps

üìñ Optimize for Readability

‚úì Do

  • ‚Ä¢ Use 16px minimum for body text
  • ‚Ä¢ Line height of 1.5-1.6 for body text
  • ‚Ä¢ Line length of 50-75 characters
  • ‚Ä¢ Sufficient contrast: 4.5:1 minimum

‚úó Don't

  • ‚Ä¢ Use text smaller than 14px for critical info
  • ‚Ä¢ Set line height below 1.4 for paragraphs
  • ‚Ä¢ Use Thin (100) weight on small screens
  • ‚Ä¢ Use light gray text on white backgrounds

üì± Mobile-First Typography

16px Input Rule

Always use at least 16px for input fields to prevent automatic zoom on iOS Safari.

Touch Target Size

Interactive elements should be at least 44x44px (iOS) or 48x48px (Android).

Responsive Scaling

Use relative units (rem, em) or responsive breakpoints for typography that adapts to screen size.

‚ôø Accessibility Guidelines

‚úì Good Contrast

Black text on white (21:1)

White text on black (21:1)

White on primary (7.2:1)

‚úó Poor Contrast

Light gray on white (3:1)

White on yellow (1.5:1)

Similar colors (2.8:1)

WCAG 2.1 Standards: Normal text needs 4.5:1 contrast ratio. Large text (18px+ or 14px+ bold) needs 3:1 minimum.
